# Claude UI 設計協作指南

這份文件整理了使用 Claude 進行介面微調時的實務原則，目標是讓產出的畫面更乾淨、精準、可落地，而不是停留在泛用的 AI 生成風格。

## 1. 協作方式

### 即時修改

把 Claude 當成設計助理，而不是一次性產圖工具。可直接用語音或文字要求它微調細節，例如：

- 替換顏色
- 調整對齊方式
- 修改按鈕或區塊間距
- 改變字距、字重或陰影強度

關鍵不是只說「變好看」，而是明確指出你不滿意的細節，讓 Claude 逐步修正。

### 暫時性行內工具

當需求涉及精準定位、裁切圖片、拖拉對齊或量測數值時，可要求 Claude 直接在頁面上建立暫時性的行內拖曳工具。

建議流程：

1. 先讓 Claude 在頁面中插入可拖曳、可量測的臨時工具。
2. 手動把位置、尺寸、裁切範圍調到理想狀態。
3. 記錄最後得到的座標與數值。
4. 要求 Claude 將這些數值寫死到正式程式碼。
5. 完成後刪除這個臨時工具，避免殘留在正式產品中。

這種方式特別適合處理難以靠語言一次描述清楚的視覺問題。

## 2. 邊框與陰影

### 避免泥濘感陰影

常見問題是使用純色邊框再疊加陰影，導致元件邊緣看起來髒、厚、糊。

### 建議做法

- 不要優先使用明顯的實線邊框。
- 改用極低透明度的深色外環來定義輪廓。
- 陰影要服務於層次，不要讓邊界更混濁。

實作方向：

- 外環可用接近 `gray-950` 的色系，透明度約 5% 到 10%。
- 陰影保持柔和，避免邊框與陰影互相打架。
- 目標是讓元件邊緣更 crisp，而不是更厚重。

## 3. 容器與截圖呈現

### 用真實素材取代假 UI

若畫面中需要展示產品內容，優先使用真實的高解析度截圖，而不是 AI 臨時生成的假介面。真實素材能直接提升可信度與完成度。

### 容器包裝原則

- 以低透明度深色背景包住截圖，建立穩定的視覺承托。
- 背景濃度要很輕，約 5% 灰色即可。
- 容器本身應維持乾淨，不要疊過多特效。

### 同心圓角

外層容器與內層圖片的圓角必須保持同心，避免出現弧度不一致的廉價感。

可執行原則：

- 先決定外層容器圓角。
- 內層圖片圓角依照外層縮減固定差值。
- 確保圖片邊界與容器弧線視覺上完全貼合。

## 4. 字體排印

### 字體選擇

- 優先使用可變字體。
- 標題可考慮使用 Display 版本以增加視覺張力。
- 若字體支援進階字型特徵，可依需求啟用，改善細節辨識度。

### 字距控制

不同尺寸文字需要不同策略：

- 大標題：略微縮緊字距，讓標題更有力量。
- 小標題或 eyebrow：改用等寬字體、全大寫、增加字距，建立資訊層級。

### 字重控制

善用可變字體提供的中間字重，不必只停留在 500 或 600。像 550 這類中間值，往往更適合精緻介面。

### 閱讀寬度

不要用固定像素限制段落寬度，優先使用字元數控制閱讀節奏。

建議：

- 段落寬度以 `40ch` 左右為基準。
- 依版型需要微調，但核心目標是讓每行字數維持舒適。

### 換行優化

適合的工具：

- `text-wrap: balance`
- `text-wrap: pretty`
- 或對應的 `text-balance`、`text-pretty` 工具類別

這類處理可減少孤字、尷尬斷行與視覺節奏不穩的問題。

## 5. 版面配置

### 打破預設置中

AI 常傾向把內容全部置中，結果容易看起來平、滿、無重點。可主動要求更有張力的非對稱版型。

例如：

- 大標題靠左，佔約 3/5。
- 說明文字與按鈕靠右，佔約 2/5。
- 讓留白和資訊密度形成對比。

這會比「標題、內文、按鈕全部置中」更成熟。

### 行內標題

可參考現代產品網站常見作法，將區塊標題與說明文字排在同一行。

建議原則：

- 標題維持主要視覺焦點。
- 說明文字使用較淡的灰階色。
- 利用同列排版提升節奏感與資訊效率。

## 6. 畫布網格與裝飾細節

### 畫布網格

如果缺少插圖或品牌視覺元素，可以透過全站一致的結構性裝飾增加精緻度，例如：

- 區塊之間的滿版水平分隔線
- 貼合容器的垂直導線
- 統一節奏的邊界框線

重點不是做出厚重框架，而是建立整體畫布感，讓頁面在空白處也有設計語言。

### Dark Shim

當圖片直接作為背景時，應在底部或文字所在區域加上暗色漸層，以維持文字可讀性。

建議做法：

- 在圖片底部覆蓋由深到透明的漸層。
- 白色或淺色文字放在漸層覆蓋範圍內。
- 不要只靠加粗字體解決可讀性問題。

## 7. 可直接套用的設計準則

### 視覺品質檢查

- 邊框是否太實、太黑、太明顯
- 陰影是否讓元件看起來混濁
- 圖片與容器圓角是否同心
- 文字層級是否只靠放大縮小，而缺少字重、字距與字體變化
- 段落是否過寬，導致閱讀疲勞
- 是否所有東西都置中，導致版面缺少節奏
- 背景圖上的文字是否有足夠對比

### Claude 指令風格

給 Claude 的指令最好具體，例如：

- 把這個按鈕外框改成 8% 透明的深色外環，不要實線邊框。
- 把標題字距稍微縮緊，字重調到 550。
- 這段文字改成 `max-width: 40ch`，並套用平衡換行。
- 將標題改為靠左 3/5，描述與按鈕放右側 2/5。
- 在圖片底部加暗色漸層，確保白字清楚。
- 幫我先做一個可拖曳裁切工具，我手動調整後再把數值寫死。

## 8. 核心結論

高品質 UI 微調的重點不是增加更多裝飾，而是更精準地控制：

- 輪廓
- 層次
- 排印
- 留白
- 對齊
- 可讀性

Claude 最適合扮演高反應速度的設計助理：先快速生成，再透過明確指令反覆打磨，必要時插入臨時工具協助量測與定位，最後把結果固化成正式程式碼。
