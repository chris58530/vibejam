# Claude UI Prompt Guide

## 1. 協作模式

### 即時修改

- 將 Claude 視為設計助理。
- 接受語音或文字微調。
- 可直接調整顏色、對齊、邊距、字距、字重、陰影。
- 以具體修改指令為主，不使用模糊描述。

### 暫時性行內工具

- 當需求涉及精準定位、裁切、拖曳或量測時，先在頁面建立 temporary inline tool。
- 工具需可拖曳、可量測、可即時回報座標與尺寸。
- 先保留為臨時實作，不納入正式設計系統。
- 使用者手動調整到理想數值後，再將數值寫死到正式程式碼。
- 數值固化後，移除 temporary inline tool。

## 2. 邊框與陰影

- 拒絕純色邊框加陰影造成的 muddy shadows。
- 不使用明顯實線邊框作為第一選擇。
- 改用極低透明度的深色 outer ring 定義輪廓。
- 建議色系接近 gray-950，透明度 5% 到 10%。
- 陰影保持乾淨、柔和、分層明確。
- 目標是 crisp，不是厚重。

## 3. 容器與截圖

- 優先使用真實高解析度產品截圖，不使用假 UI。
- 截圖外層容器使用極低透明度深色背景，建議約 5% 灰色。
- 容器視覺保持乾淨，不堆疊多餘特效。
- 內外圓角保持 concentric radius。
- 外層先定義圓角，內層圖片依固定差值縮減。
- 確保圖片弧度與容器弧度完全貼合。

## 4. 字體排印

### 字體

- 優先使用 variable font。
- 標題優先使用 Display 版本。
- 可啟用進階字型特徵以改善辨識與質感。

### 字距

- 大標題大於 24px 到 30px 時，略微縮緊 tracking。
- 小標題與 eyebrow 使用 monospace、全大寫、加寬 tracking。

### 字重

- 使用可變字體中間字重。
- 可優先嘗試 550，而不是只使用 500 或 600。

### 閱讀寬度

- 段落寬度使用 ch 單位，不使用固定 px。
- 建議預設為 max-width: 40ch。

### 換行

- 優先使用 text-wrap: balance 或 text-wrap: pretty。
- 可使用 text-balance 或 text-pretty 類別處理孤字與不自然換行。

## 5. 版面配置

- 避免所有元素預設置中。
- 優先嘗試非對稱配置。
- 主標題靠左佔 3/5。
- 說明文字與按鈕靠右佔 2/5。
- 以留白與資訊密度對比建立節奏。

### 行內標題

- 區塊標題與說明文字可排在同一行。
- 標題維持主要焦點。
- 說明文字使用 gray-600 類型的較淡灰階。
- 同列排版需保留清楚層級與節奏。

## 6. 畫布網格與裝飾

### Canvas Grid

- 在區塊邊緣加入全站一致的裝飾性邊框與分隔線。
- 水平線可滿版延伸。
- 垂直線需貼合容器。
- 裝飾目的是建立畫布感，不是增加厚重框架。

### Dark Shim

- 當圖片作為背景時，在底部或文字區域加入向上的暗色漸層。
- 白字或淺色字必須落在漸層保護範圍內。
- 不以單純加粗字體取代對比處理。

## 7. 品質檢查

- 邊框不可太實、太黑、太明顯。
- 陰影不可混濁。
- 容器與圖片圓角必須同心。
- 文字層級不可只靠字體大小區分。
- 段落不可過寬。
- 版面不可全部置中。
- 背景圖上的文字必須有足夠對比。
